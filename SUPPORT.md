# Support

Emergency Line is community software maintained on a best-effort basis. It has no paid support, uptime promise, response-time commitment, or emergency-response function.

For reproducible software problems, open a GitHub issue using the appropriate template. Remove credentials, real phone numbers, private URLs, Twilio identifiers, recordings, and personal communications before posting. Maintainers may close requests that cannot be reproduced or that depend on an unsupported provider/country/device configuration.

Use [GitHub Private Vulnerability Reporting](SECURITY.md) for security issues. Do not use GitHub issues or vulnerability reports to seek help during an actual emergency.

For billing, account verification, number availability, carrier registration, delivery errors, or provider outages, contact Twilio or the relevant carrier/provider. Maintainers cannot inspect or change a self-hoster’s account.

Always maintain a separate way to contact the owner. Call the appropriate local emergency number when immediate assistance is needed.
